# ProjectE-PolymarketTGtracker 제품 기획서 (Draft)

## 1. 한 줄 요약
스마트 지갑이 폴리마켓에서 거래하면 **즉시** 텔레그램으로 알려주는 개인용 알림 봇.

## 2. 문제 인식
- 시장에서 잘하는 지갑의 움직임은 곧바로 따라가고 싶다.
- 폴리마켓은 지갑별 공식 API가 없어, 기존 크롤링 데이터는 누락 가능성이 있다.
- 따라서 **온체인 기반 감지**로 정확하고 빠른 알림이 필요하다.

## 3. 목표 사용자
- 폴리마켓 트레이더
- 스마트 지갑을 추적하고 싶은 개인

## 4. 해결 방법 (아이디어)
- 텔레그램 봇에 지갑 주소를 등록
- 서버가 온체인 이벤트를 실시간에 가깝게 감지
- 지갑 거래 발생 시 공지 채널로 즉시 알림 발송

## 5. 핵심 가치
- **속도:** 거래 직후 알림
- **정확성:** 온체인 기반
- **간단함:** 텔레그램 명령어로 등록/관리

## 6. 사용 시나리오
1) 사용자는 텔레그램에서 `/add 0x...` 입력
2) 서버가 지갑을 저장
3) 지갑 거래 발생 → 알림
4) `/list`로 관리, `/remove`로 삭제

## 7. 제품 범위 (초기)
- 개인용(퍼스널 봇)
- 핵심 명령어: `/add`, `/remove`, `/list`, `/note`, `/help`
- 알림 내용: 지갑(별명), 이벤트명, 매수/매도, Yes/No, 가격, 수량, 시간, 폴리마켓 링크
- 공유 링크 기반의 **읽기 전용 구독** (비공개 채널 초대 링크)

## 8. 성공 기준
- 등록한 지갑 거래를 **10초 내 감지/알림** (초기 기준)
- 24시간 무중단 동작
- 중복/누락 최소화

## 9. 운영 전략 (초기)
- 폴링 10초로 시작 → 안정성/비용 모니터링 → 필요 시 더 단축 검토

## 10. 리스크
- RPC 비용 증가 가능성
- 온체인 데이터 해석 복잡도
- 네트워크/폴링 안정성

## 11. 향후 확장 아이디어
- 다중 사용자(개인별 목록)
- 웹 대시보드
- 특정 이벤트/시장만 필터링
- 트랜잭션 규모 필터
- 개인별 커스텀 목록 제공 (멀티 유저 모드)
