# MarketMotion 썸네일 디자인 시스템

## 1. 캔버스

| 항목 | 값 |
|------|-----|
| 크기 | 864 × 280 px (CSS 기준) |
| 출력 해상도 | 1728 × 560 px (2배) |
| 비율 | 약 3.09 : 1 (와이드 배너형) |

---

## 2. 레이아웃 구조

```
┌─────────────────────────────────────────────────────┐
│ [라벨]                                   [로고] │
│                                                     │
│  제목 텍스트                                        │
│  제목 텍스트                      [배경 이미지 영역] │
│                                                     │
│  ┌────────────────────┐                             │
│  │ [YES]    [NO]      │                             │
│  └────────────────────┘                             │
│                                                     │
└─────────────────────────────────────────────────────┘
```

- **좌측 영역 (약 50%)**: 제목 + YES/NO 투표 박스
- **우측 영역 (약 50%)**: 배경 이미지가 드러나는 영역
- **제목과 투표 박스는 하나의 블록**으로 묶여 좌측에서 **세로 중앙 정렬**

---

## 3. 배경 처리

### 3-1. 이미지 레이어
- 배경 이미지를 캔버스 전체에 `cover`로 깔음
- 이미지 불투명도: **70%** (배경이 충분히 보이되 제목을 방해하지 않는 선)

### 3-2. 그라데이션 오버레이
좌→우로 점점 투명해지는 어두운 오버레이를 덮음

```
왼쪽 끝    →    중간    →    오른쪽 끝
75% 불투명      30%          5% (거의 투명)
```

**목적**: 좌측 텍스트 가독성 확보 + 우측 이미지 살리기

### 3-3. 레이어 순서 (아래→위)
1. 단색 배경 (이미지 로딩 전 fallback)
2. 배경 이미지 (opacity 70%)
3. 그라데이션 오버레이
4. 콘텐츠 (제목, 투표, 로고)

---

## 4. 타이포그래피

### 4-1. 제목
| 항목 | 값 |
|------|-----|
| 폰트 | Arial (시스템 산세리프) |
| 굵기 | 900 (Black) |
| 행간 | 1.15 |
| 최대 너비 | 430px |
| 색상 | 흰색 (#FFFFFF) |

### 4-2. 자동 폰트 크기
제목 글자 수에 따라 자동 조절:

| 글자 수 | 폰트 크기 |
|---------|----------|
| 12자 이하 | 44px |
| 13~20자 | 38px |
| 21자 이상 | 32px |

### 4-3. 제목 외곽선 처리
글자가 배경 위에서 확실히 읽히도록 다중 처리:
- **텍스트 스트로크**: 1.5px 반투명 검정
- **그림자 4겹**:
  - 0 0 6px (강한 근접 그림자)
  - 0 0 12px (중간 번짐)
  - 0 2px 4px (아래 방향 입체감)
  - 0 0 30px (넓은 후광)

---

## 5. YES/NO 투표 박스

### 5-1. 전체 구조
```
┌─ wrapper (감싸는 박스) ──────────────┐
│                                      │
│  ┌──────────┐    ┌──────────┐       │
│  │   YES    │    │    NO    │       │
│  │  (58%)   │    │  (42%)   │       │
│  └──────────┘    └──────────┘       │
│                                      │
└──────────────────────────────────────┘
```

### 5-2. Wrapper 박스
| 항목 | 값 |
|------|-----|
| 배경 | 검정 35% 불투명 |
| 테두리 | 흰색 15% 불투명, 2px |
| 모서리 | 14px 라운드 |
| 안쪽 여백 | 8px 10px |
| 효과 | 배경 블러 10px + 안쪽 상단 하이라이트 |

### 5-3. 개별 버튼
| 항목 | 값 |
|------|-----|
| 크기 | 105 × 58 px (양쪽 동일) |
| 모서리 | 10px 라운드 |
| 텍스트 | YES/NO 16px 800, 퍼센트 13px 700 |

### 5-4. 우세/열세 표현 (크기가 아닌 밝기로)

**두 버튼의 크기는 항상 동일**하고, 우세 여부를 **색상 밝기와 투명도**로 구분:

| 상태 | 배경 불투명도 | 테두리 불투명도 | 글자색 | 추가 효과 |
|------|-------------|-------------|--------|---------|
| **우세** | 80% | 95% | 흰색 100% | 색상 glow (15px) |
| **열세** | 25% | 35% | 흰색 50% | 없음 |

→ 우세한 쪽은 "빛나는 느낌", 열세한 쪽은 "눌린 느낌"

---

## 6. 색상 시스템 (자동 추출)

수동으로 색을 지정하지 않음. **배경 이미지에서 자동으로 색상을 뽑아** 버튼에 적용.

### 6-1. 추출 과정
1. 배경 이미지의 **좌측 절반**에서 평균 색상 추출 (버튼이 놓이는 영역 기준)
2. 추출된 색의 **HSL 값** 분석

### 6-2. 색상 배정 규칙

| 버튼 | 색상 규칙 | 효과 |
|------|---------|------|
| **YES** | 배경 색조에서 **+150도 회전** (보색 방향) | 배경과 대비 → 눈에 띔 |
| **NO** | 배경과 **같은 색조**, 밝기만 살짝 올림 | 배경에 자연스럽게 녹아듦 |

### 6-3. 실제 적용 예시

| 배경 분위기 | 추출 색조 | YES 색상 | NO 색상 |
|-----------|---------|---------|--------|
| 어두운 남색 (배드버니) | 파랑 계열 | 빨간색 | 짙은 파랑 |
| 갈색/주황 (더운해) | 주황 계열 | 민트색 | 흙갈색 |
| 보라 (Relay) | 보라 계열 | 올리브 | 보라색 |
| 어두운 회색 (흑백요리사) | 무채색 | 녹색 | 회색 |

---

## 7. 로고

| 항목 | 값 |
|------|-----|
| 위치 | 우측 상단 고정 (top 14px, right 16px) |
| 높이 | 16px (자동 비율) |
| 불투명도 | 75% |
| 역할 | 브랜드 표시, 콘텐츠를 방해하지 않는 수준 |

---

## 8. 이미지 검색 전략

썸네일 배경에 사용할 이미지를 검색할 때 **2안을 동시에 받음**:

| 검색 방식 | 키워드 예시 | 결과 성격 |
|----------|-----------|---------|
| **A: 기본** | "Bad Bunny portrait" | 대표적/정적 이미지 |
| **B: 스마트** | "Bad Bunny Super Bowl halftime show" | 맥락적/역동적 이미지 |

- B(스마트)는 제목에서 **핵심 키워드를 조합**해서 검색
- 최소 해상도 800px 이상만 필터링
- 워터마크가 있는 스톡 사이트 자동 차단

최종적으로 A/B 두 시안을 보고 **사람이 선택**.

---

## 9. 요약: 하나의 썸네일이 만들어지는 공식

```
입력: 제목 + YES% + NO%

   ↓

1. 제목에서 검색 키워드 추출 → 이미지 2장 검색 (기본/스마트)
2. 이미지 좌측 절반에서 대표 색상 추출
3. 대표 색상 → YES(보색)/NO(동색) 자동 배정
4. 글자수 → 폰트 크기 자동 결정
5. YES%/NO% 비교 → 우세 쪽 밝게, 열세 쪽 어둡게

   ↓

출력: 2안 (기본 이미지 / 스마트 이미지)
```
