# ProjectB - Polymarket Whale Tracker (Web Widget)

## 프로젝트 개요

Polymarket에서 활동하는 고래 지갑들을 대규모로 추적하여, 거래 발생 시 실시간으로 Mm.pro 웹사이트에 피드 형태로 보여주는 위젯.

### 한 줄 요약
**고래들이 뭘 사고 파는지 실시간으로 보여주는 웹 위젯**

---

## 핵심 컨셉

### 무엇을 보여주는가
고래 지갑에서 트랜잭션이 발생할 때마다 아래 정보를 피드에 표시:

| 정보 | 예시 |
|------|------|
| 프로필 이미지 | 유저 프사 |
| 유저명 | ZhangMuZhi.. |
| 마켓명 | Will the highest temperature in Toronto be -2°C on February 11? |
| 매수/매도 | bought (초록) / sold (빨강) |
| 결과 | Yes / No / 팀명 등 |
| 가격 | at 0.99¢ |
| 수량(금액) | ($9900.00) |
| 시간 | about 4 hours ago |

### UI 참고
- KOL Whale Activity 페이지 스타일
- 스크롤 가능한 피드 (위젯 크기 고정, 안에서 스크롤)
- 최신 거래가 위에 (역순)

## ProjectE와의 차이

| | ProjectE (텔레그램 봇) | ProjectB (웹 위젯) |
|--|----------------------|-------------------|
| 용도 | 내가 직접 고른 소수 정예 추적 | 대규모 고래 모니터링 |

| 알림 방식 | 텔레그램 개인 알림 | 웹사이트 피드 위젯 |

| 성격 | 커스텀 스마트머니 추적 | 시장 전체 고래 동향 파악 |

## 데이터 현황

### 이미 수집 중인 데이터 (구글 시트)

**시트 1: 지갑 주소 목록**
- 지갑주소(address)
- 최초등장일(first_seen)
- 트위터ID(twitter_id)
- 폴리마켓 프로필 URL
- 최근 거래 정보 (이벤트명, 수량, 매수/매도, 결과, 가격, 일자)

**시트 2: 고래 활동 트랜잭션**
- 거래ID(id)
- 지갑주소(address)
- 유저명(user_name)
- 폴리마켓 프로필 URL
- 이벤트 슬러그(event_slug)
- 이벤트명(event_name)
- 매수/매도(buy_sell)
- 결과(outcome)
- 가격(event_price)
- 수량(event_size)
- 날짜(date)
- 타임스탬프(timestamp)

**시트 3: 수동으로 수집중인 주소**
- 별도로 수동 큐레이션하는 지갑 목록

## 구현 방향 (아이디어 단계)

### 데이터 흐름
```
[고래 지갑 주소 목록 (300~1,000개)]
        ↓
[온체인 모니터링 / Polymarket API 폴링]
        ↓
[트랜잭션 감지 → DB 저장]
        ↓
[Mm.pro 웹 위젯에 실시간 피드 표시]
```

### 고려 사항
- 300~1,000개 지갑 동시 모니터링 시 RPC/API 부하 관리
- 트랜잭션 감지 속도 vs 서버 비용 균형
- 기존 ProjectE의 온체인 감지 로직 참고/재활용 가능
